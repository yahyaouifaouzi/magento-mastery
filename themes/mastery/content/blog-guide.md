Looking for a specific topic? These hubs group related articles together. Each hub has a **pillar article** (the broadest overview) with **cluster articles** that dive deeper into specific areas. Start with the pillar, then explore what interests you.

## Hub 1: Core Magento 2 Architecture

Understanding Magento 2's extension system is the foundation of every custom module. This hub covers how Magento's interceptor chain works and how to hook into it properly.

### Articles

- **[Magento 2 Plugins: Before, After & Around](/blog/magento-2-plugins-before-after-around-complete-guide/)** *(pillar)* — Master the 3 interceptor types: when to use each, sort order, and real-world patterns.
- **[Complete Guide: setup:upgrade vs setup:di:compile](/blog/magento-setup-upgrade-vs-di-compile/)** — Know exactly when to run each command and why during your deployment workflow.
- **[Magento 2 Observers & Events](/blog/magento-2-observers-events/)** *(new)* — Hook into Magento's event system to react to orders, customers, and admin actions without editing core.

## Hub 2: Performance & Optimization

Speed is critical for e-commerce conversions. This hub covers Magento-specific caching, indexing, and frontend performance strategies.

### Articles

- **[Magento 2 Caching Strategies](/blog/magento-2-caching-strategies/)** *(new — pillar)* — Full-page cache, Varnish, Redis, and cache tagging: a complete caching architecture guide for Magento 2.
- **[Magento 2 Indexers Explained](/blog/magento-2-indexers-complete-guide/)** — How indexers work, mview changelog, Update on Schedule vs Update on Save, and production troubleshooting.
- **[Core Web Vitals in Magento](/blog/magento-2-core-web-vitals/)** *(new)* — Optimise LCP, INP, and CLS specifically for Magento 2 stores — from server tuning to frontend assets.

## Hub 3: Customization & API

Extend Magento 2 with custom checkout flows, payment integrations, and headless APIs. This hub covers both frontend and API-layer customisation.

### Articles

- **[Magento 2 Checkout Customization](/blog/magento-2-checkout-customization/)** *(pillar)* — Full checkout customisation: layout XML, Knockout.js components, shipping, payment, and third-party modules.
- **[Magento 2 GraphQL API: Complete Guide](/blog/magento-2-graphql-api-complete-guide/)** — Queries, mutations, schema, custom resolvers, authentication, and headless PWA patterns.
- **[REST API Best Practices](/blog/magento-2-rest-api-best-practices/)** *(new)* — Clean REST endpoints, proper error handling, bulk operations, and integration patterns for Magento 2.

---

**Not sure where to start?** If you're new to Magento 2, begin with Hub 1 — understanding plugins and DI is the prerequisite for everything else. Store owners and merchants may prefer Hub 2 first for quick performance wins. Building a custom feature? Head straight to Hub 3.
